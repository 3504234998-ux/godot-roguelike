## 三、模块一：存档系统 (P0)

### 目标

玩家关闭游戏后再次打开，能继续之前的进度。这是"完整游戏"的最低门槛。

### 功能需求

| 需求 | 说明 |
|---|---|
| 自动存档 | 每波结束后自动保存到槽位 0 |
| 手动存档 | 暂停菜单中"保存并退出"按钮 |
| 读档 | 主菜单"继续游戏"按钮（无存档时灰色不可点击） |
| 删除存档 | 设置面板中"删除存档"按钮（需二次确认） |

### SaveManager 接口设计

```gdscript
# Autoload: SaveManager
# 挂载位置: project.godot [autoload] 段

## 保存到指定槽位，返回是否成功
func save_game(slot: int) -> bool

## 从槽位读取，返回完整存档字典；槽位为空时返回 {}
func load_game(slot: int) -> Dictionary

## 删除指定槽位的存档文件
func delete_save(slot: int) -> void

## 获取槽位元数据（时间戳、等级、波次），不加载完整数据
func get_slot_info(slot: int) -> Dictionary

## 检查指定槽位是否有存档文件
func has_save(slot: int) -> bool

## 自动保存到槽位 0（由 GameManager 在波次切换时调用）
func auto_save() -> void

## 获取存档版本号，用于主菜单校验
func get_save_version(slot: int) -> String
```

### 实现要点

1. 存档文件路径：`"user://saves/save_%03d.json" % slot`（Godot 用户数据目录）
2. 使用 `FileAccess.open(path, FileAccess.WRITE)` 写入，`FileAccess.open(path, FileAccess.READ)` 读取
3. `JSON.stringify(data, "\t")` 生成带缩进的 JSON 字符串
4. 保存前校验 `GameManager.current_state == GameManager.GameState.PLAYING`
5. 加载时先调用 `GameManager.restart_game()` 清空状态，再逐个恢复属性和升级
6. 所有 I/O 操作用 `if not FileAccess.file_exists(path): return` 兜底

### 集成点

- **GameManager**: 提供 `get_save_data() -> Dictionary` 和 `restore_from_save(data: Dictionary)` 两个方法，由 SaveManager 调用
- **DataManager**: 已有 `get_weapon_data()` 等查询接口，存档只需存武器 ID，加载时重新查配置
- **主菜单**: 调用 `SaveManager.has_save(0)` 判断"继续游戏"按钮是否可用

### 测试方法

1. 开始游戏 → 玩到第 3 波 → 按 ESC → 点"保存并退出"
2. 回到主菜单 → 点"继续游戏" → 确认波次/等级/武器与退出前一致
3. 再玩几波 → 直接 Alt+F4 关闭 → 重新打开 → 点"继续游戏" → 确认自动存档生效
4. 设置面板 → 删除存档 → 返回主菜单 → "继续游戏"按钮变为灰色

## 四、模块二：天赋树 / 被动技能系统 (P1)

### 目标

玩家通过升级获取"天赋点"，在天赋树上选择被动效果，构建不同流派。天赋效果与现有升级系统互不冲突，叠加生效。

### 功能需求

| 需求 | 说明 |
|---|---|
| 天赋树 UI | 分支结构树状面板，每个节点显示名称 + 效果描述 |
| 天赋点获取 | 每 5 级获得 1 个天赋点（由 `player_level.gd` 信号驱动） |
| 天赋解锁 | 节点有前置条件（上一级节点已解锁） |
| 天赋效果持久化 | 永久被动加成，保存到存档 |
| 四条分支 | 攻击 / 防御 / 速度 / 特殊，每条 3 层 |
| 运行时叠加 | 天赋效果与升级效果叠加计算（如 天赋 +10% 伤害 * 升级 +20% 伤害） |

### 天赋树结构

```
              │               │               │
      ┌───────┼───────┐       │       ┌───────┼───────┐
  [暴击]    [穿透强化]  [攻击II] [回血]  [冲刺冷却] [拾取范围]
  10%暴击   穿透 +2    攻击 +20% 每秒0.5%  -20%冷却   +50%范围

  [致命一击]                      [冲刺无敌]
```

### TalentManager 接口设计

```gdscript
# 场景节点，挂载到 MainScene
# 通过组 "talent_manager" 被其他系统查找

## 解锁指定天赋
func unlock_talent(talent_id: String) -> void

## 检查天赋是否已解锁
func is_unlocked(talent_id: String) -> bool

## 检查天赋是否可解锁（前置条件满足 + 有天赋点）
func can_unlock(talent_id: String) -> bool

## 获取当前可用天赋点数
func get_available_points() -> int

## 获取所有已解锁天赋的 ID 列表
func get_unlocked_talents() -> Array[String]

## 获取某个天赋对指定属性的总加成（遍历已解锁天赋求和）
func get_effect_for(target: String) -> float

## 获取存档数据
func get_save_data() -> Array[String]

## 从存档恢复
func restore_from_save(data: Array[String]) -> void
```

### 集成点

- **player_level.gd**: 监听 `level_changed` 信号，每 5 级 `talent_point_gained.emit()`
- **upgrade_manager.gd**: 应用升级效果时调用 `TalentManager.get_effect_for(target)` 叠加计算
- **weapon_manager.gd**: 提供 `get_base_damage()` 等方法，天赋在最终计算中叠加
- **存档**: TalentManager 提供 `get_save_data()` / `restore_from_save()`

### UI 交互

1. HUD 上新增"天赋"按钮（或按 T 键打开）
2. 点击打开 TalentTree 面板（CanvasLayer，覆盖在游戏画面上方）
3. 可用天赋点显示在面板顶部
4. 已解锁节点亮起，可解锁节点闪烁，不可解锁节点灰显
5. 点击可解锁节点 → 确认弹窗 → 扣除天赋点 → 应用效果 → 节点变亮
6. 关闭面板回到游戏

### 测试方法

1. 进入游戏 → 升到 5 级 → HUD 应显示天赋点 +1
2. 按 T 打开天赋树 → 点击"攻击强化 I" → 确认解锁 → 伤害面板应 +10%

4. 保存退出 → 读档 → 确认天赋状态保留

## 五、模块三：Boss 技能扩展 (P1)

### 目标

当前 Boss 只有"追踪 → 冲撞"两种状态，扩展为拥有多阶段多技能的战斗，让 Boss 战更丰富。

### 当前 Boss 状态机

```
CHASE ←→ DASH_PREPARE → DASH → COOLDOWN
```

### 扩展后的状态机

```
                    ┌─── SPREAD_SHOT（环形弹幕，HP<70% 激活）
                    │
CHASE ←→ DASH_PREPARE → DASH → COOLDOWN
    │                     │
    ├── TRACKING_FIREBALL（追踪火球，每 8 秒）
    │
    ├── SUMMON_MINIONS（召唤小怪，HP<40% 激活，每 15 秒）
    │
    └── BERSERK_DASH（狂暴连冲，HP<30% 激活，连续 3 次）
```

### 新增技能详情

| 技能 | 触发条件 | 冷却 | 效果 |
|---|---|---|---|
| 环形弹幕 (SPREAD_SHOT) | HP < 70% | 6 秒 | 向四周发射 12 颗慢速子弹（子弹速度 200，伤害为 Boss 接触伤害的 50%） |
| 追踪火球 (TRACKING_FIREBALL) | 始终可用 | 8 秒 | 发射 3 颗追踪玩家的火球（子弹速度 150，有追踪转向能力，持续 5 秒后自毁） |
| 召唤小怪 (SUMMON_MINIONS) | HP < 40% | 15 秒 | 在 Boss 周围召唤 4 只 `basic_enemy`（不受波次缩放影响，使用当波基础属性） |
| 狂暴冲锋 (BERSERK_DASH) | HP < 30% | 10 秒 | 连续冲刺 3 次，每次间隔 1 秒，冲刺速度 +30%，冲刺结束进入 2 秒冷却 |

### 修改文件

| 文件 | 改动 |
|---|---|
| `scripts/enemy/boss_enemy.gd` | 添加 4 个新状态 + 对应技能逻辑 + 阶段切换判断 |
| `data/enemy_data.json` | Boss 配置增加技能参数（弹幕数量 / 火球速度等） |
| `scenes/enemy/Boss.tscn` | 可能需要增加特效 Sprite2D 节点（火球 / 弹幕预警环） |

### Boss 配置扩展 (`data/enemy_data.json` 中 boss 条目新增字段)

```json
{
  "boss": {
    "spread_shot": {
      "bullet_count": 12,
      "bullet_speed": 200.0,
      "damage_ratio": 0.5,
      "cooldown": 6.0,
      "activate_hp_ratio": 0.7
    },
    "tracking_fireball": {
      "count": 3,
      "bullet_speed": 150.0,
      "track_strength": 3.0,
      "lifetime": 5.0,
      "cooldown": 8.0
    },
    "summon_minions": {
      "count": 4,
      "enemy_type": "basic_enemy",
      "cooldown": 15.0,
      "activate_hp_ratio": 0.4
    },
    "berserk_dash": {
      "dash_count": 3,
      "dash_interval": 1.0,
      "speed_multiplier": 1.3,
      "cooldown_after": 2.0,
      "activate_hp_ratio": 0.3
    }
  }
}
```

### 实现要点

1. 每个新技能对应 `BossState` 枚举新增一个状态值
2. HP 百分比检测在 `_physics_process` 开头统一判断，切换阶段标记
3. 环形弹幕复用现有子弹对象池（`ObjectPoolManager` + `bullet_scene`），改为从 Boss 位置向四周发射
4. 追踪火球需要新建 `TrackingFireball` 子弹类型，在 `bullet_controller.gd` 中增加 `_tracking_mode` 分支
5. 召唤小怪调用 `EnemyManager` 的方法生成，不受波次权重影响
6. 狂暴冲锋复用现有 `DASH` 状态逻辑，加一个循环计数器
7. 状态机优先级：BERSERK_DASH > SUMMON_MINIONS > SPREAD_SHOT > TRACKING_FIREBALL > 普通 CHASE/DASH

### 测试方法

1. 修改 `EnemyManager.boss_spawn_time` 为 5 秒快速测试
2. 进入游戏 → 等待 Boss 出现 → 观察 CHASE 阶段正常
3. 将 Boss HP 打到 70% 以下 → 观察环形弹幕发射（12 颗子弹均匀分布）
4. 观察追踪火球每 8 秒发射 3 颗并追踪玩家
5. 将 Boss HP 打到 40% 以下 → 观察召唤小怪
6. 将 Boss HP 打到 30% 以下 → 观察连续 3 次冲刺
7. 击杀 Boss → 确认死亡掉落正常

## 六、模块四：地图场景 / 障碍物 (P2)

### 目标

从空旷场地升级为有墙壁、障碍物的地图，增加走位策略和空间感。

### 功能需求

| 需求 | 说明 |
|---|---|
| 地图边界 | 用墙壁围住可移动区域（矩形，约 3000×3000 像素） |
| 随机障碍物 | 石头、柱子等，阻挡子弹和移动，每次开局随机生成 |
| 碰撞处理 | 玩家和敌人被障碍物阻挡（CharacterBody2D.move_and_slide 自动处理） |
| 子弹碰撞 | 子弹碰到障碍物销毁（不穿透），穿透子弹碰到障碍物消耗穿透次数 |
| 视觉风格 | 暗色地板 + 深灰墙壁，使用 TileMap 实现 |

### 节点结构


### MapGenerator 接口设计

```gdscript
# 挂载到 MainScene 的 Node 节点

## 生成完整地图（边界墙 + 随机障碍物）
func generate_map() -> void

## 清除当前地图
func clear_map() -> void

## 获取所有障碍物的全局坐标（供存档 / 调试）
func get_obstacle_positions() -> Array[Vector2]

@export var map_width: float = 3000.0
@export var map_height: float = 3000.0
@export var obstacle_count: int = 15       # 随机障碍物数量
@export var obstacle_min_size: float = 40.0
@export var obstacle_max_size: float = 120.0
@export var safe_zone_radius: float = 250.0 # 玩家出生点周围无障碍物
```

### 实现要点

1. 使用 Godot 4 的 `TileMapLayer`（非 TileMap 旧版），在编辑器中创建暗色地板 tileset
2. 边界墙壁由 4 段 `StaticBody2D` 组成矩形框，每段有 `CollisionPolygon2D`
3. 随机障碍物：在 `map_width × map_height` 范围内随机放置，使用 `RandomNumberGenerator`
4. 玩家出生点 (0, 0) 周围 `safe_zone_radius` 范围内不生成障碍物
5. 障碍物生成后检查与其他障碍物最小距离（至少 80px），避免堆叠
6. 如果没有 TileSet 资源，先用纯色 `ColorRect` 或 `Polygon2D` + `StaticBody2D` 实现功能，视觉效果后期补充

### 测试方法

1. 进入游戏 → 确认四面有墙壁围住 → 玩家无法走出地图
2. 确认地图内有随机障碍物，玩家/敌人无法穿过
3. 子弹击中障碍物时销毁
4. 重新开始 → 障碍物位置变化（随机种子不同）
